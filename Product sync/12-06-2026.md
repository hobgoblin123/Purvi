### Tool integrations

**Ponytail** (github.com/DietrichGebert/ponytail) — code-minimalism plugin, installed for Claude Code. Adopted as a During-building gate ("Minimal-code ladder"). Rationale: Purvi enforces minimalism at the product level (should this exist?), ponytail at the code level (how little code once it does). Two explicit local overrides recorded in the gate and snapshot:
1. Stable external dependencies remain acceptable — the ladder orders where to look first, it does not ban libraries.
2. The ladder governs *what* gets written, never the process — TDD is not waived.

**Graphify** (github.com/safishamsi/graphify) — knowledge-graph tool, was already installed locally (CLI + skill) but undocumented in Purvi. Snapshot added; the existing Ground-truth gate gained one clause: if `graphify-out/` exists, query the graph before grepping/re-exploring. Knowledge graphs are being built for the active projects this session (rollout status in to-do's below).

**Graphs stay local (Nithin's call, reasoned through):** `graphify-out/` is gitignored per project, never pushed. The graph is a derived artifact that can embed environment facts (paths, service names, doc excerpts) — the same class of detail scrubbed before backups — and it regenerates on demand. Same logic as ground truths: machine-local, not shared-repo content. (Upstream's "commit the graph" default is for multi-dev teams; doesn't apply here.)

How the pieces relate: graphify feeds the *before building* side (understand what exists), ponytail governs the *during building* side (write the minimum). Both serve the same thesis — most waste is building the wrong thing or too much of it.
